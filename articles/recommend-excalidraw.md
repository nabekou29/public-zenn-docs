## Excalidraw とは

[Excalidraw](https://excalidraw.com/) は、手書き風のダイアグラムを作成できるオープンソースの作図ツールです。

![excalidraw-intro](/images/recommend-excalidraw/excalidraw-intro.png)

私は Obsidian の中で Excalidraw を使っているので、以降はそこでの使い方を前提に紹介します。

ここでは導入方法などは割愛しますが、プラグインを入れればすぐに試すことができます。

## Excalidraw のおすすめポイント

図を描いたりするためのツールはたくさんあります。ペイント系のツールや、Figma、Draw.io、など。
中でも Excalidraw をすすめたいポイントは以下の通りです。

- **自由度が高いが、テキトーに作ってもそれっぽく見える**

  プリセットと基本図形を組み合わせれば簡単に見やすい図が作れます。
  プリセットのカラーパレットがあるが、それとは別に自由に色を選べるなど、バランスが良いなと思います。
  少し凝った図を作りたいときにも困らない点も良いです。

- **ローカルで動作する**

  Obsidian の中で動作していて、インターネット接続がなくても使えます。
  クラウドにデータを預けたくない場合にも安心ですし、アカウント登録なども不要です。

- **プラグイン (scripts) で機能拡張が可能**
  グラフを生成させたり、整列させたり、いろいろあります。後で紹介しますが自作もできます。
  たぶん Obsidian 版の Excalidraw のみの機能です。

## なんで手書き風の図解が必要なのか

私は記事を書くにあたって常々思っていたのですが、技術記事は全体的に堅くなりがちだなと感じてます。
文章やコードは仕方ないと思うのですが、図解すらも堅いとなんだか読んでて疲れるような、そんな感覚があります。

単に柔らかい雰囲気が好きと言うのもありますが、実際、**伝えたい情報によってトーンを変えるのは読み手にとってもわかり易いと思う** のです。
考えていること・未確定なことは手書き風に、確定情報はかっちりした図にするなどすると、視覚的にも情報がスッと入ってきやすくなると思います。

でも結局、使いたい一番の理由は **かわいいから** です。

## プラグイン (scripts) とちょっとAIの話

Excalidraw は scripts と呼ばれるもの機能を拡張することができます。

JS で書けるので、普段から JS を書いている人なら簡単に作れると思います。

嘘です。

内部のデータ構造や API を理解するのにはどうしても労力がかかります。なので AI に書かせましょう。  
Claude Code は Excalidraw の scripts を書くのに非常に役立ちました。

AI はこのようなプラグイン的なものを作るのに、めちゃくちゃ向いてます。
